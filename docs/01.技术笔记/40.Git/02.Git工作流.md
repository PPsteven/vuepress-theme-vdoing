### 分支命名

```
feature：为程序添加新功能。
fix：为程序修复新发现的 bug。
test：为程序添加新的测试覆盖。
refactor：整理重构程序的代码，提高可维护性。
shrink：移除程序过时不用的代码。
update: 更新描述性内容，比如更新文档、更新代码版本号、更新配置文件等。
```
