从输入 baidu.com 到显示网页，后台发生了什么？
序号 步骤 协议 对应层级 详细步骤
1 输入游览地址 游览器解析URL地址：
url: http://user:pass@www.example.com:80/dir/index.html?uid=1#id1
   - 协议名称: http
   - 登录信息
   - 服务器地址
   - 服务器端口号(http默认80,https默认443)
   - 文件路径
   - 查询字符串(GET)
   - 片段标识符(前端路由)
2 申请外网IP DHCP 应用层 向 DHCP 服务器(一般是路由器)发送 DHCP 请求，网络服务供应商会自动分配IP地址
UDP 传输层 双方都没有IP地址是无法建立TCP连接，故使用UDP。DHCP协议中规定发送方端口是67，接收方端口是68
IP 网络层 DHCP协议规定，发送方的IP地址就设为 0.0.0.0，接收方的IP地址设为255.255.255.255。
MAC(ARP) 网络层靠下 查找下一跳 MAC地址的过程是：1. ARP缓存 2. ARP广播请求，等待ARP回应 
3 游览器查找域名的IP地址 DNS 应用层 DNS 查找过程：
1. 游览器缓存：游览器会缓存DNS记录
2. 系统缓存：检查本地硬盘的 hosts 文件
3. 路由器缓存
4. 本地DNS缓存/ISP DNS缓存：本地DNS服务器(一般由ISP提供) ，提供是递归查询
5. 本地DNS搜索DNS：ISP DNS --> 根服务器 --> .com 域名服务器-->baidu.com 域名服务器，提供的是迭代查询
UDP 传输层 UDP无需建立连接，开销小，故DNS报文封装成UDP数据报
OSPF(IP) 应用层 路由选择协议，通过OSPF维护路由表
4 浏览器给web服务器发送一个HTTP请求 TCP 传输层 通过三次握手建立TCP连接，默认端口号80
HTTP 应用层 默认发送 http 数据报
5 baidu服务器重定向 HTTP 应用层 返回3XX重定向：baidu.com => 301 重定向 到 https://www.baidu.com
                                www.baidu.com => 307 重定向 到 https:// www.baidu.com
6 服务器处理返回HTTP响应 HTTP 应用层 状态码：2XX,3XX,4XX,5XX 等
TCP 传输层 四次握手断开连接
7 前端渲染 HTML  HTTP 应用层 发送HTTP请求获取嵌入的 图片、音频等资源。发送Ajax 异步请求。
